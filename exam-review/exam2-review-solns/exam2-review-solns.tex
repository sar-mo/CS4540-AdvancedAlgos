\subsection*{\Exercise}
Short-answer questions. No justification required.
\begin{enumerate}
    \item Consider Fortune’s algorithm to compute the Voronoi diagram of n sites in
    the plane. What is the maximum number of arcs that any one site can contribute to the
    beach line?
    \item Which of the following assertions about the Delaunay triangulation (DT) of
    a set $P$ of $n$ points in the plane are true? Select all that apply.
    \begin{enumerate}[label=\roman*.]
    \small
        \item The closest pair of points are connected by an edge in the DT.
        \item The farthest pair of points cannot be connected by an edge in the DT.
        \item Among all triangulations of P, the DT maximizes the minimum angle.
        \item Among all triangulations of P, the DT minimizes the maximum angle.
        \item Among all triangulations of P, the DT minimizes the total sum of edge lengths
    \end{enumerate}
    \item Recall the \textsf{MaxExact3SAT} problem. The input is a Boolean formula in conjunctive normal form such that every clause is the disjunction of exactly three literals and you wish to output an assignment of the variables that satisfy the maximum number of clauses. True or false: there exists an $\epsilon$-approximation for the \textsf{MaxExact3SAT} problem for any $\epsilon > 0$.
    \item True or false: there exists a 2-approximation for the \textsf{TSP} problem.
\end{enumerate}

\subsection*{Solution}
    \begin{enumerate}
        \item The maximum number of arcs that any one site can contribute to the beach line is $n$. This occurs when each site (excluding the first) splits an arc originating from the first site.
        \item The correct choices are i and iii.
        \item False. The lecture notes give a categorization of many \textsf{NP-Complete} problems in terms of their approximability. There we can see that \textsf{MaxSAT} is approximable for some but not all $\epsilon$. A proof is given on \href{https://en.wikipedia.org/wiki/MAX-3SAT}{Wikipedia}.
        \item False if we assume it's non-metric \textsf{TSP}. If we assume it is \textsf{Metric-TSP}, then the answer is true. A 2-approximation for \textsf{TSP} is given in the notes:

        \vspace{2mm}
        Find a minimum spanning tree $T$ of $G$. Double every edge of $T$ to obtain a Eulerian graph. Find a Eulerian tour $\mathcal{T}$ on this graph. Output the tour that visits the vertices of $G$ in the order of their first appearance in $T$. Let $\mathcal{C}$ be this tour.

        \vspace{2mm}
        Proof of correctness: Note that $|T| \le \text{OPT}$  because deleting an edge from an optimal tour yields a spanning tree. Moreover, $|\mathcal{T}| = 2 \cdot |T|$. Because of the triangle inequality, $|\mathcal{C}| \le |\mathcal{T}|$. Hence, $|\mathcal{C}| \le 2 \cdot \text{OPT}$.
    \end{enumerate}

    \vspace{3mm}

In frustration, you decide to give your students full credit if every diagram they submit is the Voronoi diagram of some point set. But how do you tell?\footnote{I had a similar issue as a TA for \href{https://github.com/sar-mo/CS2051-HonorsDiscreteMath/blob/main/sp23/hw-supplements/hw3-supp/images/colored_map2.pdf}{2051}.}

    \vspace{3mm}
    Describe and analyze an algorithm to determine, given a planar straight-line graph G, whether $G$ is a Voronoi diagram. Your algorithm should either return a finite point set $P$ such that $G$ is the Voronoi diagram of P, or report correctly that no such point set exists. [\textit{Hint:} Consider the case of four sites.]

\subsection*{Solution\footnote{from \href{https://arxiv.org/pdf/1308.5550.pdf}{here}.}}
    % https://ieeexplore.ieee.org/document/6605976
    % https://arxiv.org/pdf/1308.5550.pdf
    For a single intersection of the Voronoi diagram, you will generally have 3 edges, and 3 sectors between the edges. Call the sectors (and their angles) $A$, $B$, and $C$. Also, call the edge between sectors $A$ and $B$ the edge $ab$, and likewise for edges $bc$ and $ca$ (see Figure \ref*{fig:givp})

    \vspace{3mm}
    There should be an original site point within each of these sectors; let site a be the site in sector $A$, site $b$ in sector $B$, and site $c$ in sector $C$. Note that the angles to the sites on either side of a sector boundary must be equal, because the distance from the Voronoi edge to each site must be equal. For example, the angle from site $a$ to edge $ab$ must be the same as the angle from edge $ab$ to site $b$; call this angle $X$. Likewise let angle $Y$ be the angle from site $b$ to edge $bc$ and from $bc$ to site $c$; and $Z$ the angle from $c$ to $ca$ and from $ca$ to $a$. This gives you the equations:
    \begin{align*}
        A &= Z + X \\
        B &= X + Y \\
        C &= Y + Z
    \end{align*}
    With the solution (simplified because $A + B + C = 2\pi$:
    \begin{align*}
        X &= (A + B - C)/2 = \pi - C \\
        Y &= (B + C - A)/2 = \pi - A \\
        Z &= (C + A - B)/2 = \pi - B
    \end{align*}
    This gives you a ray from any Voronoi intersection to each of its 3 sites. And the intersection of the rays from neighboring Voronoi intersections to the same cell site will give you a location for that site.

    \vspace{3mm}
    If you only have 3 sites, then you can only have one Voronoi intersection. In that case, you won't be able to determine your sites -- just their angles from the intersection. In all other general cases, you can find at least one site as described above; reflection across Voronoi edges should then determine the location of all other sites, including extremal cells that have only one Voronoi intersection.

      \item \texttt{create(R)}: create an empty data structure to handle an \( R \)-by-\( R \) grid.
      \item \texttt{insert(i,j)}: insert the pixel \( (i,j) \) into the data structure, where \( i \) and \( j \) are integers between \( 0 \) and \( R-1 \).
      \item \texttt{find(x,y)}: return the inserted pixel which is closest to \( (x,y) \) in Euclidean distance, where \( x \) and \( y \) are integers between \( 0 \) and \( R-1 \).
    \end{itemize}
    
    The \texttt{create} and \texttt{insert} operation should take \( \mathcal{O}(R^2) \) time; the \texttt{find} operation should take \( \mathcal{O}(1) \) time. For partial credit, give an algorithm that takes \( \mathcal{O}(NR^2) \) time for \texttt{insert}, where \( N \) is the number of points already inserted into the data structure. (\textit{Hint:} think simple.)
    
    \begin{enumerate}[label=(\alph*)]
      \item Describe how to implement \texttt{create(R)}. Indicate what is being stored.
      \item Describe how to implement \texttt{find(i,j)}.
      \item Describe how to implement \texttt{insert(i,j)}.
    \end{enumerate}

\subsection*{Solution}
    \begin{enumerate}
        \item \texttt{create($R$)}. Use a helper data type \texttt{Pixel} to manipulate pixels. Initialize an $R$-by-$R$ grid of pixels to \texttt{null} so that \texttt{nearest[$i$][$j$]} is the inserted pixel closest to $(i,j)$.

        \item  \texttt{insert($x, y$)}. Create a new \texttt{Pixel} object $p$ with coordinates $(x, y)$. For each $i$ and $j$, check whether $(i,j)$ is closer to $p$ than it is to \texttt{nearest[$i$][$j$]}. If it is, update \texttt{nearest[$i$][$j$]}.
    \end{enumerate}

\subsection*{\Exercise}
    Prove that Chebyshev's inequality is tight: in other words, find a probability distribution for $X$ and a value $a$ such that
    \begin{equation*}
        \Pr{|X - \E{X}| \geq a} = \frac{\Var{X}}{a^2}.
    \end{equation*}
    \textit{Hint:} This random variable will take only three values.

\subsection*{Solution}
    Let $a = 1$ and $X$ be a random variable such that $$p_{X}(x) = 
    \begin{cases}
        \frac{1}{2} & \text{if $x = -1$}, \\
        \frac{1}{2} & \text{if $x = 1$}, \\
        0 & \text{otherwise.}
    \end{cases}$$
    Then $\E{X} = 0$ and $\Var{X} = 1$, so $$\Pr{|X - \E{X}| \geq a} = \Pr{|X| \ge 1} = 1$$ and $$\frac{\Var{X}}{a^2} = 1.$$

\subsection*{\Exercise}
    As in the homework, consider the general problem where we have $m$ machines $M_1 \ldots M_m$ and a set of $n$ jobs. Each job $j$ has a processing time $t_j$. We seek to assign each job to one of the machines so that the loads placed on all machines are as "balanced" as possible. That is, if $A(i)$ are the set of jobs assigned to machine $M_i$ , then the load for machine $M_i$ is total time required.  \[ T_i = \sum_{j \in A(i)} t_j\] We want to minimize the $makespan$ $T = \max_i T_i$.
    \begin{enumerate}
        \item Describe a set of infinite set of jobs such that the makespan of the greedy assignment is exactly $(2 - 1/m)$ times the makespan of the optimal assignment, where $m$ is the number of machines.
        \item  Describe an efficient algorithm to solve the minimum makespan scheduling
        problem exactly if every processing time $T_{i}$ is a power of two.
    \end{enumerate}

\subsection*{Solution}
    \begin{enumerate}
        \item Consider the following instance: $(m^2 - m)$ jobs with unit processing time and a single job with processing time $m$. Suppose the greedy algorithm schedules all the unit jobs before the long job, then the makespan of the schedule obtained is $(2m - 1)$ while the optimal makespan is $m$. Hence, the algorithm gives a schedule which has makespan $\left(2 - \frac{1}{m}\right)$ times the optimal.
        \item First, sort the set of jobs in decreasing order of their processing time. Since every processing time $T_{i}$ is a power of two, we can do this in linear time using radix sort. Then, we can run the greedy algorithm on this sorted set of jobs.
    \end{enumerate}

\subsection*{\Exercise}
    It is a fact that we still have no closed-form expression for the tail of the Normal distribution, which must be computed by numerically evaluating the integral. Let \(X \sim \mathcal{N}(0,1)\). Your goal is to produce upper bounds on \(P(X \geq a)\), where \(a > 0\).
    \begin{enumerate}
        \item Use Markov's inequality to bound \(\Pr{X \geq a}\). Note: This is not as trivial as it might seem because \(X\) is not non-negative. It will help to observe that:
        \[
        \Pr{X \geq a} = \Pr{X \geq a \mid X > 0} \cdot \Pr{X > 0}.
        \]
        Now define the non-negative r.v.\ \( Y = [X \mid X > 0] \) and note that \(\Pr{X \geq a \mid X > 0} = \Pr{Y \geq a}\).
        \item Use Chebyshev’s inequality to bound \(\Pr{X \geq a}\).
        \item \textit{(Optional)} Derive the Chernoff bound for \(\Pr{X \geq a}\).
    \end{enumerate}

\subsection*{Solution}
    \begin{enumerate}
        \item Define the non-negative r.v.\ \( Y = [X \mid X > 0] \). Then 
        $$\begin{aligned}
            \Pr{X \geq a} &= \Pr{X \geq a \mid X > 0} \cdot \Pr{X > 0} \\
            &= \Pr{Y \geq a} \cdot \Pr{X > 0} \\
            &\le \frac{\E{Y}}{a} \cdot 0.5 \\
        \end{aligned}$$
        We can calculate $\E{Y}$ as
        $$\E{Y} = 2\int_{0}^{\infty} x\frac{1}{\sqrt{2\pi}} e^{-\frac{x^{2}}{2}} \ dx = \sqrt{\frac{2}{\pi}}.$$ Thus $$\Pr{X \geq a} \le \frac{\sqrt{2/\pi}}{2a} = \frac{1}{\sqrt{2\pi}a}.$$
        \item Chebyshev's inequality states that \[ \Pr{|X - \E{X}| \geq a} \le \frac{\Var{X}}{a^{2}} \]

        \vspace{2mm}
        In our case, \( X \sim \mathcal{N}(0,1) \). Therefore, Chebyshev's inequality becomes: \[ \Pr{|X| \geq a} \leq \frac{1}{a^2} \] Since \( X \) is symmetric about its mean (zero), we can write: \[ \Pr{X \geq a} = \frac{1}{2}\Pr{|X| \geq a} \]

        \vspace{2mm}
        Therefore, applying Chebyshev's inequality gives:
        \[ \Pr{X \geq a} = \frac{1}{2}\Pr{|X| \geq a} \leq \frac{1}{2} \cdot \frac{1}{a^2} = \frac{1}{2a^2} \]
        \item See \href{http://prob140.org/fa18/textbook/chapters/Chapter_19/04_Chernoff_Bound}{here} for the solution.
    \end{enumerate}

\subsection*{\Exercise}
    Previously in this class, we discussed the \textbf{coupon collector} problem (do you remember when?) As a reminder: there are \( n \) distinct types of coupons you would like to collect. Each day you are sent a random coupon from among the \( n \) types. Let \( X \) denote the number of days needed to collect all \( n \) distinct coupons, given that coupons are chosen randomly with replacement. The following (famous) identity\footnote{known as the \href{https://en.wikipedia.org/wiki/Basel_problem}{Basel problem}.} is useful in answering some of the questions below:
    \[ \sum_{i=1}^{n} \frac{1}{i^2} = \frac{\pi^2}{6}. \]
    \begin{enumerate}
        \item What is \( \E{X}? \)\footnote{Of course we already discussed this in class, but good to practice!} What does this approach for high \( n \)? Write your answer using \( \mathcal{O}(\cdot) \).
        \item Derive \( \Var{X} \). What does this approach for high \( n \)? Write your answer using \( \mathcal{O}(\cdot) \).
        \item Derive an asymptotic upper bound on \( \Pr{X \geq 2n \ln(n)} \) for large \( n \) using Markov’s inequality.
        \item Derive an asymptotic upper bound on \( \Pr{X \geq 2n \ln(n)} \) for large \( n \) using Chebyshev’s inequality. Express your answer using \( \mathcal{O}(\cdot) \).
    \end{enumerate}
    Note: For \( \E{X} \) in (c) and (d), use the asymptotic mean from part (a).

\subsection*{Solution}
    \begin{enumerate}
        \item Let $X_{k}$ denote the number of days needed to collect $k$ distinct coupons, given that you have already collected $k - 1$ distinct coupons. After collecting $k - 1$ distinct tokens, the probability you will collect another distinct token is $\frac{n - k + 1}{n}$. Thus $X_{k} \sim \text{Geo}(\frac{n - k + 1}{n})$, so $\E{X_{k}} = \frac{n}{n - k + 1}$. By the linearity of expectations, $$\E{X} = \sum_{k = 1}^{n} \E{X_{k}} = \sum_{k = 1}^{n} \frac{n}{n - k + 1} = nH_{n} \sim \mathcal{O}(n\ln(n)).$$
        \item Once again, let \(X_k\) be the number of days after the \((k - 1)\)\textsuperscript{th} distinct coupon to collect the \(k\)\textsuperscript{th} distinct coupon. 
        Its variance is:
        \[
        \Var{X_k} = \frac{1 - p}{p^2} = \frac{1 - \frac{n - k + 1}{n}}{\left(\frac{n - k + 1}{n}\right)^2} = \frac{n(k - 1)}{(n - k + 1)^2}
        \]
        Given that each \(X_k\) is independent,
        $$\begin{aligned}
            \Var{X} &= \sum_{k=1}^{n} \Var{X_k} \\
            &= \sum_{k=1}^{n} \frac{n(k - 1)}{(n - k + 1)^2} \\
            &= n \sum_{k=1}^{n} \frac{n - k}{k^2} \\
            &= n^2 \sum_{k=1}^{n} \frac{1}{k^2} - n \sum_{k=1}^{n} \frac{1}{k} \\
            &= n^2 \sum_{k=1}^{n} \frac{1}{k^2} - nH_n.
        \end{aligned}$$
        We simplify this using the Basel Problem to obtain $$\Var{X} = n^2 \left(\frac{\pi^2}{6}\right) - nH_n \sim \mathcal{O}(n^{2})$$

\subsection*{\Exercise}
    Let $X$ be the number of fixed points of a random permutation on $n$ elements. Calculate $\E{X}$ and $\Var{X}$ and use these to bound the probability that the number of fixed points of a random permutation is at least $10$. Are Chernoff bounds useful to improve this?

\subsection*{Solution}
    First, we write $X$ as the sum of indicators, $X = \sum_{i=1}^n I_i$, where $I_i$ is $1$ when element $i$ in the permutation is a fixed point. In a random permutation, $i$ is equally likely to be in any location, so the probability that it is in its own position $\Pr{I_i = 1} = \E{I_i} = 1/n$. Thus by linearity of expectation, $\E{X} = n\E{I_1} = 1$.

    \vspace{3mm}
    Now, these $I_i$ are not independent random variables, so we cannot add the variances. To start, we consider $\E{I_iI_j}$ for $i \neq j$. $I_iI_j$ is $1$ if both $i$ and $j$ are fixed points. There are $n(n-1)$ possible places for this pair to go, all equally likely, thus $\E{I_iI_j} = \frac{1}{n(n-1)}$. Thus:
    \begin{align*}
        \E{X^2} &= \E{\left(\sum_i I_i\right)^2}\\
        &= \E{\sum_i I_i^2 + \sum_{i\neq j} I_iI_j}\tag{By expanding the square}\\
        &= \sum_i \E{I_i^2} + \sum_{i\neq j} \E{I_iI_j} \tag{By Linearity of
        Expectation}\\
        &= \sum_i \frac1n + \sum_{i\neq j} \frac{1}{n(n-1)} \\
        &= \frac{n}{n} + \frac{n(n-1)}{n(n-1)} = 2
    \end{align*}
    Thus $\Var{X} = \E{X^2} - \E{X}^2 = 2-1^2 = 1$.
    \begin{itemize}[label={\textbf{--}}]
        \item Markov's inequality gets us that $\Pr{X \geq 10} \leq 1/10$.
        \item Chebychev's gets us that $\Pr{X \geq 10} \leq \Pr{|X - 1| > 9} \leq 1/81$.
        \item Chernoff bounds cannot be used, since the $I_i$ are not independent.
    \end{itemize}

    \begin{enumerate}
        \item The \textbf{chromatic number} $\chi(G)$ of an undirected graph $G$ is the minimum number of colors required to color the vertices, so that every edge has endpoints with different colors. Computing the chromatic number exactly is \textsf{NP-hard}, because \textsf{3Color} is \textsf{NP-hard}.

        Prove that the following problem is also \textsf{NP-hard}: Given an arbitrary undirected graph $G$, return any integer between $\chi(G)$ and $\chi(G) + 473$.
        \item Let $\alpha(G)$ denote the number of vertices in the largest independent set in a graph $G$. Prove that the following problem is \textsf{NP-hard}: Given a graph $G$, return any integer between $\alpha(G) - 31337$ and $\alpha(G) + 31337$.
    \end{enumerate}
    [Note: This does not contradict the possibility of constant \textbf{factor} approximation algorithms.]
    
\subsection*{Solution}
    \begin{enumerate}
        \item First note that computing $\chi(G)$ is \textsf{NP-hard} by a reduction from \textsf{3Color}, since
        \begin{align*}
            \exists \ \text{3-coloring of $G$} &\rightarrow \chi(G) \le 3 \\
            \not\exists \ \text{3-coloring of $G$} &\rightarrow \chi(G) \ge 4
        \end{align*}
        Now let \( G' \) be the union of 473 copies of \( G \), with additional edges between \textit{every} vertex of each copy and \textit{every} vertex in \textit{every} other copy. Given \( G \), we can easily build \( G' \) in polynomial time by brute force. Let \( \chi(G) \) and \( \chi(G') \) denote the minimum number of colors in any proper coloring of \( G \), and define \( \chi(G') \) similarly.

        \vspace{3mm}
        \(\Longrightarrow\) Fix any coloring of \( G \) with \( \chi(G) \) colors. We can obtain a proper coloring of \( G' \) with \( 473 \cdot \chi(G) \) colors, by using a distinct set of \( \chi(G) \) colors in each copy of \( G \). Thus, \( \chi(G') \leq 473 \cdot \chi(G) \).

        \vspace{3mm}
        \(\Longleftarrow\) Now fix any coloring of \( G' \) with \( \chi(G') \) colors. Each copy of \( G \) in \( G' \) must use its own distinct set of colors, so at least one copy of \( G \) uses at most \( \left\lceil \frac{\chi(G')}{473} \right\rceil \) colors. Thus, \( \chi(G) \leq \left\lceil \frac{\chi(G')}{473} \right\rceil \).

        \vspace{3mm}
        These two observations immediately imply that \( \chi(G') = 473 \cdot \chi(G) \). Thus
        \begin{align*}
            \exists \ \text{3-coloring of $G'$} &\rightarrow \chi(G') \le 3 \cdot 473 \\
            \not\exists \ \text{3-coloring of $G'$} &\rightarrow \chi(G') \ge 4 \cdot 473,
        \end{align*}
        so given an algorithm returning any integer between $\chi(G)$ and $\chi(G) + 473$, we could compute \textsf{3Color} in polynomial time.

        \vspace{3mm}
        \textcolor{ForestGreen!80}{Note: You can also do this from a reduction from computing $\chi(G)$:}
        
        \vspace{3mm}
        \textcolor{ForestGreen!80}{
        ``If we could compute some integer \( k \) such that \( k \leq \chi(G') \leq k + 473  \), then \( \chi(G) = \chi(G')/473 = \left\lceil k/473 \right\rceil \). Thus, if we could compute such an integer \( k \) in polynomial time, we could compute \( \chi(G) \) in polynomial time. But computing \( \chi(G) \) is \textsf{NP-hard!}"}
        
        \item Let $G'$ be the union of 31337 copies of $G$, with additional edges added as follows: for every edge $uv$ in $G$, add an edge from \textit{every} copy of $u$ to \textit{every} copy of $v$.

        \vspace{3mm}
        \(\Longrightarrow\) For any independent set $S$ in $G$, we can select the copies of every vertex $v \in S$ in $G'$ to construct a independent set 31337 times larger. Thus $\alpha(G') \ge 31337 \cdot \alpha(G)$.

        \vspace{3mm}
        \(\Longleftarrow\) The largest independent set in $G'$ can at most have a representative from each vertex of the largest independent set in $G$ from each of the 31337 copies. Hence, $\alpha(G') \le 31337 \cdot \alpha(G)$.
        
        \vspace{3mm}
        These two observations imply that \( \alpha(G') = 31337 \cdot \alpha(G) \). If we could compute some integer \( k \) such that \( k - 31337 \leq \alpha(G') \leq k + 31337 \), then \( \alpha(G) = \alpha(G')/31337 = \left\lceil k/31337 \right\rceil \). Thus, if we could compute such an integer \( k \) in polynomial time, we could compute \( \alpha(G) \) in polynomial time. But computing \( \alpha(G) \) is \textsf{NP-hard!}
    \end{enumerate}

The objective is to route the calls so as to minimize the total load on the network.
